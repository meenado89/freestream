📚 COMPLETE BUILD GUIDE — HOW TO BUILD YOUR OWN STREAMING SITE

How a Free Streaming Site Works

A free streaming site like this doesn't need to host videos itself. It acts as a directory and player that embeds videos from other legal free sources — just like how a blog embeds a YouTube video.

Here's the basic flow:

  1. Find free/legal movies — Use public domain archives and Creative Commons sources (see Tab 2)
  2. Build your pages — HTML for structure, CSS for design, JavaScript for interactivity
  3. Embed the videos — Use <iframe> or <video> tags pointing to free sources
  4. Add AdSense ads — Google pays you every time a visitor sees or clicks an ad
  5. Host the site — Deploy on GitHub Pages, Netlify, or Vercel (all free)

You are essentially a curator, not a video host. This is 100% legal when you only use public domain or Creative Commons-licensed content.

Where to Find Free, Legal Movies

These are the BEST sources — all free, all legal, and all embeddable:

  • Internet Archive (archive.org) — Largest free movie library on the internet. Hundreds of thousands of public domain films. Filter by "Moving Images" → use their embed codes directly. ✅
  • Wikimedia Commons — CC-licensed short films, documentaries. Direct MP4 links available.
  • Vimeo (CC filter) — Go to vimeo.com/creativecommons and filter by license. Many filmmakers allow free embedding.
  • YouTube (CC filter) — On YouTube, filter search by "Creative Commons" license. You can embed these freely.
  • Blender Open Movie Project — Blender Foundation produces stunning animated films (Sintel, Big Buck Bunny, Elephants Dream) all under CC license with full HD.
  • Open Culture (openculture.com) — Curated list of 1,150+ free legal films. Great starting database.
  • PublicDomainMovies.net — Classic Hollywood films (pre-1927) that are now in the public domain.
  • Fmovies / Tubi API approach — Advanced: Tubi has a free tier. You can build on top of their content API if approved.
✅ Public Domain — Free forever ⚠️ CC License — Check terms ❌ Copyrighted — Never embed

How to Embed Videos on Your Page

There are 2 main ways to play videos:

Method 1 — Embed from Internet Archive (easiest):
Every Archive.org video has an embed code. Go to the video page → click "Share" → copy the <iframe> code. Paste it into your HTML.

Method 2 — Direct MP4 with HTML5 <video> tag:
If you have a direct .mp4 URL (from Archive.org or Wikimedia), use:

<video controls width="100%" poster="thumbnail.jpg">
  <source src="https://archive.org/download/movie.mp4" type="video/mp4">
  <track kind="subtitles" src="subs.vtt" srclang="en" label="English">
</video>

Method 3 — YouTube Embed (for CC YouTube videos):

<iframe width="100%" height="400"
  src="https://www.youtube.com/embed/VIDEO_ID"
  allowfullscreen></iframe>

The <video> tag approach gives you the most control — you can add subtitles (.vtt files), custom controls, and ad overlays on top.

How to Monetize with Ads

There are 3 layers of ads you can run:

  1. Google AdSense (Banner Ads) — Sign up at adsense.google.com. Once approved, paste a small JavaScript snippet into your HTML. Google auto-places ads in your banners. You earn per 1,000 views (RPM ~ $1–$5 for general content).
  2. Google AdSense / Video Ad Networks (Pre-roll Ads) — For video ads before movies start, you need a Video Ad Network. Options: Google IMA SDK (free, integrates with your HTML5 video player), SpringServe, or VDO.AI for higher CPMs on video.
  3. Affiliate Links — Add links to streaming services (Netflix, Disney+) using their affiliate programmes. Earn commission when visitors sign up.

How to add a Google AdSense banner:

<!-- Paste this in your <head> -->
<script async src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"
  crossorigin="anonymous"></script>

<!-- Banner unit -->
<ins class="adsbygoogle"
  data-ad-client="ca-pub-XXXXXXXXXXXXXXXX"
  data-ad-slot="XXXXXXXXXX"
  data-ad-format="auto"></ins>
<script>(adsbygoogle = window.adsbygoogle || []).push({});</script>

How to Host & Launch Your Site

You don't need to pay for hosting to start. These are all free:

  • GitHub Pages — Push your HTML/CSS/JS to a GitHub repo, enable Pages in settings. Your site goes live at yourusername.github.io/yourrepo. Free forever.
  • Netlify (netlify.com) — Drag and drop your folder. Site is live in 30 seconds. Free SSL, custom domain support, 100GB bandwidth/month free.
  • Vercel (vercel.com) — Same as Netlify but slightly more developer-focused. Great for when you move to React.

Step-by-step to go live today:

  1. Create a free account at Netlify.com
  2. Build your site locally — index.html + style.css + script.js
  3. Drag your project folder onto Netlify's deploy area
  4. Your site is LIVE with an https URL
  5. Connect a custom domain (e.g. streamfree.com) — buy from Namecheap ~$10/year

For SEO: Add a sitemap.xml, meta tags for each movie, and submit to Google Search Console.

Legal & Copyright — What You Must Know

IMPORTANT: Never stream copyrighted movies without a licence. This is illegal in most countries.

Here's what IS legal:

  • Public Domain films — Films where copyright has expired (generally pre-1928 in the US). You can host, embed, and monetise these freely.
  • Creative Commons licensed films — Films whose creators chose a CC licence. Read each licence — some allow commercial use (CC BY), some don't (CC NC).
  • Films you have direct permission for — If a filmmaker grants you a licence to host their film, get it in writing.

Add these pages to your site:

Privacy Policy (required for AdSense) Terms of Service DMCA Takedown Policy Cookie Consent Banner

Use a free generator like TermsFeed.com or Iubenda.com for these legal pages.

What to Learn — Your Roadmap

Since you already know HTML, CSS, JS, and Bootstrap 5 — you're ready to build this right now. Here's what else helps:

  • Phase 1 (Now) — Build a static site: pure HTML + CSS + JS. Use fetch() to load movie data from a local JSON file. This is enough for a working v1.
  • Phase 2 (1–2 months) — Learn JSON APIs. Fetch movie data from TMDB API (free) for posters, ratings, descriptions. No backend needed.
  • Phase 3 (3–4 months) — Learn basic Node.js + Express to build a simple backend. Add user accounts, watchlists, ratings.
  • Phase 4 (6+ months) — Learn React or Vue for a full SPA (Single Page Application) experience like Netflix.

Free APIs to use RIGHT NOW:

TMDB API — Movie posters, ratings, info Archive.org API — Search public domain films YouTube Data API — Embed CC films Open Subtitles API — Multi-language subs
